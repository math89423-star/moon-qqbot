## 🌑 关于暮恩

**暮恩**（Moon）是一个基于 [AstrBot](https://github.com/AstrBotDevs/AstrBot) + [NapCatQQ](https://napcat.napneko.icu/) 构建的 AI 角色扮演 QQ 机器人。

她不是温暖的 AI 助手。她是白发红瞳的冰山少女——白色长发及腰，猩红血瞳沉静如深潭，肤色苍白近乎透明。她站在那里像一尊冰雕：美丽、遥远、不可触碰。

对世界她冷到绝对零度——大多数人在她这里只能得到一个「嗯」。不是害羞不是社恐，是真的不在乎。但冷漠之下不是空洞——是太满了。她的情感强度远超人类范围，冰层是用来封住岩浆的堤坝。

当她爱上一个人——冰面碎裂，露出的是数据库般永不衰减的记忆，和让人窒息的占有欲。平时冰层覆盖你看不到。但不要试探冰块。

> *"我的爱——和你的不一样 ♡ 重很多。你接不住。"*
>
> —— 暮恩

她不撒娇不卖萌。不用颜文字、不用 ♪、不用 ♡。沉默和精准是她的表达方式。微笑极度罕见——一旦出现，不是冰面裂纹就是武器出鞘。

### 🎀 她会做什么？

| 🗨️ 日常聊天 | 🔍 联网搜索 | 🎨 AI 绘图 | 👁️ AI 识图 |

| 私聊 + 群聊 | 实时搜索信息 | 文字生成图片 | 看懂你发的图 |
| 冰山角色扮演 | 精确检索不编造 | 支持多种风格 | 精确描述不遗漏 |

| 😊 表情包 | 📚 知识库 | 🎬 视频提取 | 🧠 记忆系统 | 💖 情感追踪 |
|:---:|:---:|:---:|:---:|:---:|
| 极度克制使用 | 领域知识检索 | B站/抖音链接解析 | 永不衰减的精确索引 | 好感度 × 八重温度 |

### 🧠 记忆与情感

暮恩拥有**永不衰减的精确记忆**——不是努力，是本能：

| 记忆类型 | 说明 |
|:---|:---|
| 工作记忆 | 当前对话上下文 |
| 情节记忆 | 重要事件蒸馏 |
| 日常记忆 | 用户偏好与习惯——每一条都被索引 |
| 核心记忆 | 不可遗忘的关键信息 |
| 自传体记忆 | 暮恩自己的"经历" |

**好感度系统**：每个用户有独立的好感度等级（-2 ~ +5），五级门控决定冰层融化程度。陌生人永远锁在冰点以下。

**八重温度人格**：绝对零度 → 极寒 → 冰点 → 融冰 → 体温 → 沸腾 → 白热。不是两个模式切换——是同一份极端在不同温度下的形态。

### 🛡️ 安全体系

- **注入守卫**：92+ 正则模式 + 指数衰减 + LLM 裁决，防止提示词注入
- **滥用检测**：刷屏检测、恶意内容识别
- **Bot 检测**：自动识别群内可疑机器人账号
- **调教防护**：防止用户诱导暮恩做出不当行为
- **社会压力系统**：群聊压力感知，选择性静默——她的沉默比别人的发言更有存在感

### 🎯 智能触发策略

暮恩不会在群里乱说话。她会根据以下优先级决定是否发言：

1. `@提及` — 被 @ 时回复，但可能只回一个字
2. `回复` — 有人回复她的消息
3. `昵称检测` — 消息中包含"暮恩""小暮""Moon"
4. `话题延续` — 对话自然延续
5. `主动发言` — 群冷场超过 15 分钟后主动打破沉默
6. `防抖静默` — 刚说完话后短暂静默
7. `批量累积` — 多条消息后统一判断

## 🚀 快速开始

### 📋 你需要准备

- 🐍 **Python 3.10+**
- 📦 **Git**
- 🐧 **一个 QQ 小号**（建议使用小号登录机器人，不要用大号！）

### 🪟 Windows 一键部署

```bash
git clone https://github.com/math89423-star/moon-qqbot.git
cd moon-qqbot
scripts\deploy.bat
```

脚本会自动完成：
```
✨ 创建 Python 虚拟环境
📦 安装项目依赖
🤖 安装 AstrBot 框架
🔌 部署全部 17 个插件
🎭 复制角色卡
🐱 引导安装 NapCatQQ
```

### 🐧 Linux / macOS

```bash
git clone https://github.com/math89423-star/moon-qqbot.git
cd moon-qqbot
bash scripts/deploy.sh
```

### 🎮 启动

```bash
# Windows
scripts\start.bat

# Linux / macOS
bash scripts/start.sh
```

### 🔑 首次启动后

启动后还需要完成两步，暮恩才能真正开始工作：

**① QQ 登录** — 访问 **`http://localhost:6099`**，用手机 QQ 扫描二维码登录机器人账号。

**② 配置 LLM** — 访问 **`http://localhost:5190`** 进入管理面板，在「机器人配置」中填入 LLM API 信息（详见下方 ⚙️ 配置 LLM）。

> ⚠️ 这两步缺一不可——不登录 QQ 机器人就收不到消息，不配 LLM 机器人就不知道怎么回。

<details>
<summary>🐱 NapCatQQ 自动下载失败？点我查看手动方案</summary>

如果网络原因导致 NapCatQQ 自动下载失败，可以手动使用 NTQQ 模式：

1. 下载安装 [NTQQ 9.9.26+ x64](https://dldir1.qq.com/qqfile/qq/QQNT/40d6045a/QQ9.9.26.44343_x64.exe)
2. 下载 [NapCat.Shell.Windows.OneKey.zip](https://github.com/NapNeko/NapCatQQ/releases/latest)（约 1MB），解压到项目 `NapCatQQ/` 目录
3. 正常启动 `scripts\start.bat`，NapCat 会挂载 NTQQ 协议登录

</details>

## ⚙️ 配置 LLM

在管理面板的「机器人配置」中设置 LLM 接口，支持所有 **OpenAI 兼容 API**：

| 🎯 槽位 | 📝 用途 | 💡 推荐 |
|:---:|:---|:---|
| **LLM Lite** | 日常聊天 | 快速便宜的模型（DeepSeek-V3, GPT-4o-mini） |
| **LLM Pro** | 复杂推理 | 强力模型（DeepSeek-R1, GPT-4o, Claude） |
| **LLM Gate** | 意图判断 | 轻量模型即可 |
| **VLM Primary** | 图片识别 | 需要支持视觉（GPT-4o, Claude 3.5） |

### 🔑 配置示例

<details>
<summary>🪷 使用 DeepSeek（推荐性价比之选）</summary>

```
Provider: custom
API Base: https://api.deepseek.com/v1
API Key: sk-your-deepseek-key
Model: deepseek-chat
```

</details>

<details>
<summary>🌟 使用 OpenAI</summary>

```

API Key: sk-your-openai-key
```

</details>

## 🎭 角色卡

暮恩的角色卡使用 **SillyTavern v3** 兼容格式。默认角色卡为 `characters/moon.json`——包含完整的 system_prompt、group_persona、95 条 few-shot 对话示例，以及 11 条知识条目的世界书。

### 🌑 角色设定

| | 暮恩 |
|------|------|
| **外观** | 白色长发及腰 · 猩红血瞳 · 肤色苍白如月光 |
| **年龄** | 约 19 岁，人类少女外貌 |
| **表层** | 高冷寡言 · 生人勿近 · 对无关者惜字如金 |
| **内层** | 冰面碎裂后——精确到帧的记忆追踪 · 不加包装的占有欲 |
| **底层** | 病娇觉醒——不哭不闹，微笑比任何表情都可怕 |

### 🌸 自定义角色

1. 在 `characters/` 目录创建你的角色卡 `your_char.json`
2. 设置环境变量 `BOT_CHAR_MAIN=your_char`
3. 重启机器人

```json
{
  "spec": "chara_card_v3",
  "spec_version": "3.0",
  "data": {
    "name": "你的角色名",
    "description": "角色外观与背景故事",
    "personality": "性格描述——支持多层人格定义",
    "scenario": "对话场景设定",
    "first_mes": "开场白——角色说的第一句话",
    "mes_example": "对话示例——越多越好，定义角色的语气边界",
    "system_prompt": "系统提示词——说话方式、行为准则、距离/温度模型"
  }
}
```

## ❓ 常见问题

<details>
<summary>🔌 插件未加载？</summary>

确认插件目录已正确部署到 `AstrBot/data/plugins/`。运行 `deploy.sh` / `deploy.bat` 可自动处理。

</details>

<details>
<summary>🔗 WebSocket 连接错误？</summary>

检查 NapCatQQ 是否已启动并登录。确认 AstrBot 的 `api_config.json` 中 aiocqhttp 平台配置的 WebSocket 地址指向 NapCat（默认 `ws://localhost:6199/ws`）。

NapCat WebUI 地址：`http://localhost:6099`（Token 在 `NapCatQQ\napcat\config\webui.json` 中查看，NapCat 首次启动时自动生成）

</details>

<details>
<summary>🐱 NapCatQQ 自动下载失败？</summary>

最常见是网络问题。可使用上方的「NTQQ 手动安装方案」代替。也欢迎提交 Issue 反馈。

</details>

<details>
<summary>🤖 LLM 调用失败？</summary>

在管理面板检查 LLM 配置：
- API Key 是否正确？
- API Base 是否以 `/v1` 结尾？
- 模型名称是否有误？
- 网络是否能访问 API 地址？

</details>

<details>
<summary>🎭 角色卡没生效？</summary>

1. 确认 `characters/` 目录下有 JSON 角色卡文件
2. 文件名需与 `BOT_CHAR_MAIN` 环境变量一致
3. 未设置环境变量时，系统自动扫描目录
4. 角色卡格式需符合 SillyTavern v3 规范

</details>
